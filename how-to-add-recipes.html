How to Add Recipes

Official, locked process for adding recipes to the site.

Overview

  • Each recipe lives in its own HTML file
  • Each recipe has a corresponding entry in recipes.json
  • Sidebar behavior depends on exact markup and IDs
  • Printing depends on print.css
  • Categories must use standardized plural names to avoid sidebar duplication

Pre-Flight Checklist

  • Start from recipe-template.html — never from scratch
  • Do not add emojis anywhere (HTML or JSON)
  • Do not change CSS or JS paths
  • Ensure all ingredients include quantities
  • Ensure procedures are numbered steps

Category Standards (Locked)

All recipe categories are plural. Categories represent collections, not individual items. Using singular forms will cause duplicate categories to appear in the sidebar.

Use only the standardized categories below:

  • Appetizers
  • Beans
  • Beverages
  • Breads
  • Desserts
  • Main Dishes
  • Reference
  • Sauces
  • Sauces & Dressings
  • Sauces & Seasonings
  • Side Dishes
  • Snacks
  • Soups & Stews
  • Vegetables

Do not invent new categories without updating documentation, normalization scripts, and validation rules.

Step 1: Create the Recipe HTML

  1. Copy recipe-template.html
  2. Rename using kebab-case (example: golden-spice-drink.html)
  3. Edit only text content — not structure

Never change:

  • <main class="layout">
  • Sidebar markup or IDs
  • Class names
  • Script include paths

Required Page Sections (Order Matters)

  1. Page header (title, subtitle, badges)
  2. Navigation bar
  3. Ingredients
  4. Procedure
  5. Optional notes
  6. Footer metadata

Printing Rules

  • All recipes must link print.css
  • Print output includes only:
    • Title
    • Ingredients
    • Procedure
  • Sidebar and decorative elements are suppressed

Add Entry to recipes.json

{
  "title": "Recipe Title",
  "slug": "recipe-slug",
  "category": "Category Name",
  "path": "recipe-slug.html",
  "author": "Author Name",
  "source": "Source Description"
}

Final Validation

  • Sidebar loads and search works
  • Print button produces clean output
  • No console errors
