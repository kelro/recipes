How to Add Recipes

Official, locked process for adding recipes to the site.

Overview

  • Each recipe lives in its own HTML file.
  • Each recipe has a corresponding entry in recipes.json.
  • All pages must use recipe-template-new.html.
  • Asset paths are always root-relative (css/, js/).

Pre-Flight Checklist

  • Copy recipe-template-new.html
  • Rename using kebab-case (example: golden-spice-drink.html)
  • Edit content only â€” never structure
  • No emojis in HTML or JSON
  • All ingredients must include quantities
  • Procedures must use numbered steps

Locked Structure Rules

  • Do not modify <main class="layout">
  • Do not modify sidebar markup or IDs
  • Do not change CSS or JS paths
  • Do not change class names
  • All Recipes link must be href="./"

Required Page Sections (Order Matters)

  1. Page header (title, subtitle, badges)
  2. Navigation bar
  3. Ingredients card
  4. Procedure card
  5. Optional Notes card
  6. Footer metadata

Category Standards (Locked)

All categories are plural. Use only these:

  • Appetizers
  • Beans
  • Beverages
  • Breads
  • Desserts
  • Main Dishes
  • Reference
  • Sauces
  • Sauces & Dressings
  • Sauces & Seasonings
  • Side Dishes
  • Snacks
  • Soups & Stews
  • Vegetables

Printing Rules

  • All recipes must link print.css
  • Print button must use onclick="window.print()"
  • Sidebar is automatically hidden in print view

Add Entry to recipes.json

{
  "title": "Recipe Title",
  "slug": "recipe-slug",
  "category": "Category Name",
  "path": "recipe-slug.html",
  "author": "Author Name",
  "source": "Source Description"
}

Git Workflow

git add recipe-name.html
git commit -m "Added recipe-name"
git push

Final Validation

  • Sidebar loads and search works
  • Print output is clean
  • No console errors
  • No incorrect asset paths
